# Glossary

## In Plain English

This glossary is here for both technical and non-technical readers.

## Terms

### Digital Product Passport

A structured digital record about a product. In this system it can include product identity, technical details, compliance-related fields, files, public views, and machine-readable exports.

### Passport Type

A database-backed product passport definition used by the app. It determines what fields exist for a type of product.

### Passport Module

A code-defined template for a passport type. Modules live in backend source code and can be seeded into the database.

### Semantic Model

A machine-readable dictionary or vocabulary that gives stable meaning to passport fields.

### Compliance Profile

A rule bundle attached to a passport module. It can require certain header fields, semantic mappings, company identifiers, facilities, or carrier policies.

### DID

Decentralized Identifier. In this app, DID documents are exposed by the backend for companies, product subjects, and facilities.

### Public Viewer

The separate app that opens released public passport URLs without the logged-in dashboard shell.

### Repository

The company file store used for documents, uploads, and symbols referenced by passports.

3. Semantic Class Graph

Define reusable classes, class-to-class properties, controlled enums, explicit IRIs, and SHACL cardinalities.

The semantic class graph is required. Build the first layer once, then refine it without removed properties being recreated. Clear Semantic Graph only when you intentionally want to unlock and rebuild that first layer. Linked entries that remain in the graph keep their metadata synchronized when Part 2 changes. Linked scalar fields can use their Part 2 datatype or override it with a controlled enum without becoming custom properties. Choose “Custom” only for concepts that are not represented by module sections or fields. RDFS domain and range come from the owning class and selected range; SHACL min/max counts enforce cardinality.
